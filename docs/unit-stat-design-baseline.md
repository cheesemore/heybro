# 非首领单位数值设计：基准表与生成流程

本文档约定「未来新增非首领单位」时的配表流程与乘区规则。

**重要**：下文两张基准表为**内嵌在本文档中的固定数值**，是设计的权威来源。仓库里的具体兵种配置可能被删除、改名或废弃；**配表与评审一律以本 MD 中的数字为准**，不必再依赖某个 JSON 键名是否存在。

---

## 一、基准值表（快照）

### 1. 近战设计基准：兽人战士（敌方）

用于坦克 / 刺客等**以近战为参照**的定位。以下为一组完整示例数值。

| 属性 | 数值 | 说明 |
|------|------|------|
| 碰撞半径（设计像素） | 44 | 战场代币 / 碰撞用 |
| 生命 | 270 | |
| 攻击 | 26 | 单次普攻（表底=战场值；近战基准） |
| 攻击间隔（秒/次） | 0.62 | 数值越小出手越快；与战场 `attackInterval` 同义 |
| 射程 | 10 | 低于 `RANGED_ATTACK_RANGE_THRESHOLD`（100）视为近战 |
| 移速 | 505 | |

**基础篇乘积**：`生命 × 攻击 ÷ 攻击间隔`  
本快照：`270 × 26 ÷ 0.62 ≈ 11323`（取整仅作对比用）。

## 二、术语

| 术语 | 含义 |
|------|------|
| **强势属性** | 该单位相对同类基准更突出、允许偏高的那一维（如坦克的「生命」、刺客的「攻击」）。 |
| **弱势属性** | 该单位相对同类基准更保守、允许偏低的那一维（如坦克的「攻击或间隔」、刺客的「生命」）。 |
| **弱特技** | 对局面影响偏小的额外机制。 |
| **强特技** | 对局面影响显著、需占用更多预算的额外机制。 |
| **负面词条** | 对单位自身明显不利的词条（易伤、自损、短腿等），需在数值上补偿。 |

## 三、生成流程

### 步骤 1：按定位选基准，并标定强/弱属性

1. **坦克**  
   - 以**第一节「近战设计基准」表中的数值**为参照。  
   - **生命**相对该基准**上浮**；**攻击**或**攻击间隔**向另一侧调整，使整体仍贴近模型（可用「生命 × 攻击 ÷ 攻击间隔」做 sanity check）。  
   - 建议：**强势属性 = 生命**；**弱势属性 = 攻击或攻击间隔**（择一为主弱项）。

2. **刺客**  
   - 仍以近战向基准为参照时，以**第一节表**为锚。  
   - **攻击**上浮；**生命**下降。  
   - 建议：**强势属性 = 攻击**；**弱势属性 = 生命**。

3. **远程或辅助**  
   - 以**第二节「远程设计基准」表中的数值**为起点。  
   - **提高射程** → **生命**倾向**下降**。  
   - **降低射程** → **生命**倾向**上升**。  
   - **辅助**：在远程基准之上，更多通过**牺牲攻速**（拉长攻击间隔）来换机制空间，再微调生命与攻击。  
   - 建议：为每个单位明确写出**强势属性**与**弱势属性**（可为生命、攻击、攻击间隔、射程、移速中的组合，但需一眼可读）。

### 步骤 2：特技与负面词条的数值乘区

在步骤 1 得到的基础表之上，对**弱势属性**与**强势属性**做下列修正（多特技时**叠加**）：

| 条件 | 规则 |
|------|------|
| 拥有 **弱特技** | **弱势属性**再乘 **(1 − 10%) = 0.9**（每项弱特技叠乘一次，见下）。 |
| 拥有 **强特技** | **弱势属性**再乘 **(1 − 20%) = 0.8**（每项强特技叠乘一次）。 |
| 拥有 **多个特技** | 弱特技、强特技分别按个数**连乘**上述因子（例：1 弱 + 1 强 → 弱势属性 ×0.9×0.8）。 |
| 拥有 **负面词条** | **强势属性**乘 **(1 + 30%) = 1.3**（多条负面词条是否叠乘未指定时，默认**仅乘一次 1.3**；若你日后规定可多条叠加，在此文档更新即可）。 |

> **说明**：「弱势属性减少 10%」按**属性当前值 ×0.9** 理解（不是扣绝对点数）。若一条单位有多个弱势维度，需在设计案中标明特技修正作用在**哪一维**（或按比例分摊），避免歧义。

## 四、与工程实现的对齐（非权威，仅便于落地）

若仓库里仍有小怪/盟友配置表，常见约定如下；**与第一节、第二节数字冲突时，仍以本文档两表为准**。

- 配置里常把「秒/次」写在名为 `attackSpeed` 的字段中，进战场后映射为攻击间隔。  
- 游戏内「远程」判定阈值（射程 ≥ 100 等）以实现代码为准；第二节表里射程 **235** 按远程讨论。

## 五、魔兽副本用书表（怪物 + 章节）

以下文件由脚本从 `docs/reference-classic-vanilla-wow-roguelike-level-design.json` 生成，**数值以脚本启发式 + 第一节/第二节快照为准**；生成后可直接手改纠错。

| 文件 | 内容 |
|------|------|
| `src/game/config/wowBookMonsters.json` | **唯一小怪数值与命名源**（`monsters[]`）。含 `dungeonId: "legacy"` 的 12 条为原 `enemies.json` 模板兵种，id 与战场 `enemyPaint` 一致。字段含 `baseMaxHp`、`baseAtk`、`attackSpeed`（秒/次）等。 |
| `src/game/config/wowBookChapters.json` | `chapters[]`：章节、`monsterGroup`、`finalBoss`。 |
| `src/game/config/wowBookBosses.json` | `bosses[]`：关底首领元数据 + **战斗基准**（`hitRadius`、`baseMaxHp`、`baseAtk`、`attackSpeed`、`range`、`moveSpeed`、`skillIds`）；战场读取见 `resolveWowBookBossCombat`。 |

**立绘文件**：用书怪可选 PNG 路径为 `public/assets/wow-mobs/<与 monsters[].id 完全相同>.png`；详见 `public/assets/wow-mobs/README.txt`。未放置时仍用 `public/assets/enemies/<enemyPaint>.png` 模板图。

**改数值的约定**：小怪只改 **`wowBookMonsters.json`**；`baseAtk` 即战场攻击，**不要**在业务代码里再乘全局攻或近战攻倍率。

**关底首领数值**：以 **`wowBookBosses.json` 对应当前章节 `chapterIndex` 那一行** 为表底；缺字段时用 **`WOW_BOOK_BOSS_TABLE_DEFAULT`**（`wowBookData.ts`，与生成脚本 `DEFAULT_BOOK_BOSS_COMBAT` 一致）。进场生命以 `baseMaxHp` 表底直接走 `scaledEnemyHp`（表内已含原×10 池化）；攻击表底 `baseAtk` **直接**作为首领战缩放基准（无运行时攻击倍率）。`skillIds` 为空则白板仅普攻。

重新生成（会覆盖上述 JSON）：`npm run gen:wow-book`
