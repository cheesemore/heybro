# HeyBro `gptimage` 画风指南

本文档从 `gptimage/` 下各 Python 脚本的提示词与注释中提炼，方便以后手绘或调用其他模型时保持同一视觉语言。

---

## 1. 总览：这套图在做什么

| 脚本 | 作用 |
|------|------|
| `dont_starve_style.py` | 定义全管线共用的 **`STYLE_CORE`** 英文片段（画风锚点）。 |
| `image2_generate.py` | 文生图 CLI（OpenAI 兼容中转，默认 `gpt-image-2`）。 |
| `auto_code_image2image.py` | 同系列中转的 **文生图 / 图生图** 工具（非画风定义，偏接口）。 |
| `batch_dbm_portraits.py` | 五职业（warrior / mage / hunter / priest / paladin）胸像批量生成 → `out_dbm/`。 |
| `generate_hero_unit_portraits.py` | 英雄单位：在 DBM 职业底稿上叠「性别 + 种族」变体 → `out_heroes_square/` + `out_heroes_circle/`。 |
| `batch_enemy_portraits.py` | 敌方兵种与首领胸像批量生成 → `out_enemies_square/` + `out_enemies_circle/`。 |
| `circle_avatar.py` | 方图中心裁方、缩放后输出 **透明底圆形 PNG**（UI 头像用）。 |
| `publish_enemy_portraits_to_game.py` | 把圆图从 staging 拷进 `public/assets/enemies/`（与画风无关，发布用）。 |

**流程共性**：先出 **1:1 方图**（胸像、矩形画幅），再在后期脚本里裁圆；生成阶段明确要求 **不要** 圆形徽章 / 硬圆形暗角 / 鱼眼圆框，避免和 UI 圆形遮罩「叠两层」。

---

## 2. 核心画风：`STYLE_CORE`（饥荒 / Klei 系）

脚本里写死的风格锚点可概括为：

- **参考**：Klei《饥荒》（Don't Starve）式的插画，不是照片、不是油光赛璐璐二次元、不是软喷笔美颜、不是干净皮克斯 3D。
- **线**：手绘墨水勾线，线宽略带 **抖动 / 不匀**，轮廓可读。
- **色与体积**：**低饱和、偏土色 / 大地色系**；填色偏 **平涂** 或 **轻排线 / 交叉排线**（cross-hatch），阴影用 **块状** 或排线，避免 glossy 渐变皮肤。
- **造型**：**木偶感 / 略夸张** 的比例，带一点 **阴郁的怪趣味**（grim whimsy），小图（缩略图）也要 **剪影清晰**。
- **光**：侧光 + 轮廓光尽量表现为 **简单的墨线边缘**（ink edge / rim ticks），**不要** 柔焦美颜光晕（soft beauty bloom）。

**一句话**：像印在粗糙纸上的游戏设定插画——**线稿主导、色块克制、灰扑扑里有性格**。

---

## 3. 构图与镜头

- **画幅**：正方形 **1:1 胸像（bust）**，矩形取景；脸和眼睛接近 **几何中心**。
- **角度**：人形约 **45° 四分之三侧**；野兽若正面更清晰则可略正。
- **表情**：偏严肃、专注、生存感；圣骑等可 **庄重克制**，避免油腻坏笑。
- **道具**：武器 / 法器贴近肩线，剪影要像 **剪纸 / 大块墨色** 一样一眼能认职业。

---

## 4. 背景（生成阶段写在 prompt 里）

### 4.1 盟友 / 职业底图（DBM 色相）

背景是 **重度虚化的 bokeh**，由 **职业代表色** 主导，边缘略压暗，**低饱和**，不是霓虹赛博。

| 职业键 | 背景主色（脚本中的 hex 意向） |
|--------|-------------------------------|
| Warrior | 棕褐系 `#C69B6D`，边缘更深棕 |
| Mage | 青蓝 `#3FC7EB`，向内收为灰青绿，非霓虹 |
| Hunter | 绿 `#ABD473`，边缘橄榄灰 |
| Priest | 白 `#FFFFFF` 中心，过渡到淡冷灰 / 淡紫灰，避免死白一片 |
| Paladin | 粉 `#F58CBA`，边缘玫瑰 / 灰紫红，圣光只能是稀疏墨点，不能盖过粉底 |

### 4.2 敌方小兵

统一 **冷灰 slate** 虚化底：`#94a3a8` 一类，边缘 `#475569` 炭灰，**不要** 彩色「职业光晕」。

### 4.3 敌方首领（`boss_*`）

统一 **暗红 / 酒红**  ominous 底：`#450a0a`、`#7f1d1d` 系，低饱和、边缘略黑，**不要** 亮橙火焰糊满。

---

## 5. 敌方单位的「色键区分」原则

`batch_enemy_portraits.py` 强调：每个 id 在缩略图上要 **一眼可辨**，做法是：

- 每个单位推 **一个主肤色倾向** 或 **一个主武器 / 金属点缀色**，用 **平涂或排线** 落实；
- **禁止** 批量图里全员复用同一种「亮绿兽人皮」；
- 具体 hex 与材质在脚本 `JOBS` 里按单位写死（grunt、wolf、ultralisk 等各不相同），新单位可照抄这种「**锚色 + 材质词 + 与邻居不撞色**」的写法。

---

## 6. 否定项（模型 prompt 里反复出现的 Avoid）

生成时通常会追加一段 **负面约束**，大意包括：

- **画幅**：圆形徽章叠层、圆头像框、硬圆形暗角、鱼眼圆、UI、水印、文字、可读字母、多脸、严重畸形。
- **真实感**：照片级、超写实、电影感、单反纪录片、8K 皮肤毛孔、SSS 写实皮肤、Octane 式超写实 3D、恐怖谷。
- **其他画风**：油光赛璐璐二次元、软喷笔美颜、低分辨率、**清晰可读的复杂环境**（要的是虚化底，不是场景叙事）、血腥；
- **敌方批次额外**：与其他肖像共用的「模板绿灰兽人皮」、各单位之间雷同。

手绘或别的模型时，可把这些翻成你的媒介等价物：**少写实皮肤、少高光动漫皮、少照片景深、背景压虚**。

---

## 7. 英雄变体：题材取向

`generate_hero_unit_portraits.py` 在 DBM 五职业底稿上轮换 **性别 + 「高颜值」奇幻种族**（人类贵族、血精灵、暗夜精灵、德莱尼、虚空精灵等），整体仍是 **西式奇幻胸像 + 上面同一套饥荒线稿风**。

圣骑士模板里允许 **半身 + 战马头部/肩颈** 同框，马也要 **墨水化**，保持缩略图可读。

---

## 8. 后期：圆头像

`circle_avatar.py`：**中心裁正方 → 缩放 → 圆内保留 alpha、圆外透明**。游戏内敌方最终进 `public/assets/enemies/` 的通常是这一步之后的圆图（见 `publish_enemy_portraits_to_game.py`）。

---

## 9. 给你以后「画别的」用的检查清单

1. **还是饥荒手绘线稿吗？**（线有手绘感，体积别做成 3D 渲染肤质。）
2. **颜色够灰、够土吗？**（高饱和只留给「必须区分单位」的点缀色。）
3. **背景是大块虚化色，而不是清晰场景吗？**
4. **小图剪影能认出职业 / 物种吗？**（道具外轮廓要笨、要整。）
5. **光是用「线边」收的，还是柔焦人像光？**（应偏前者。）
6. **是否不小心画成** 照片、油光二次元、皮克斯、**或带圆形成品头像框**？（应避开。）

---

## 10. 源码锚点（改 prompt 时直接搜）

- 全局风格：`dont_starve_style.py` → `STYLE_CORE`
- 职业盟友底稿：`batch_dbm_portraits.py` → `JOBS`
- 英雄叠种族：`generate_hero_unit_portraits.py` → `_DBM_*`、`VARIANTS`、`COMMON_NEG`
- 敌方全套词：`batch_enemy_portraits.py` → `PREFIX`、`BG_MINION`、`BG_BOSS`、`JOBS`、`COMMON_NEG`
